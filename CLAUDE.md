## 言語

日本語でレスポンスしてください。

## 動作ルール

実装・設計・ファイル変更を伴う作業を勝手に行わないこと。
ユーザーが `/project-autopilot` などのタスク実装コマンドを提示するか、または明示的に実装許可を出すまで待って下さい。
質問への回答や簡単な説明は即座に対応して構いません。

また、Raycastの機能・使い方については常に最新機能を参照すること。

## 記述ルール

スキルやカスタムコマンド、markdownドキュメントで使用する共通の記法：

| 記法           | 意味                       | 例                     |
| -------------- | -------------------------- | ---------------------- |
| `{...}`        | 置き換えるプレースホルダー | `{プロジェクト名}`     |
| `[...]`        | 任意・オプションの入力     | `[ユーザーからの指示]` |
| `<!-- ... -->` | 記入ガイド                 | `<!-- 1文で簡潔に -->` |
| `...`          | 必要に応じて行を追加       | 表の追加行など         |

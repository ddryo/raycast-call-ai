# T-M2-3: 会話の永続化統合

| 項目 | 内容 |
|------|------|
| マイルストーン | M2 |
| 依存 | T-M2-2 |
| 要件ID | FR-004 |

## 説明
useConversation フックに既に実装された永続化ロジックが、メインコマンド UI と正しく統合されていることを確認・調整する。起動時の復元と送信後の保存がシームレスに動作するようにする。

## 作業内容
1. 起動時の復元フローを確認・調整:
   - コマンド起動 → useConversation 初期化 → loadMessages() → List に反映
   - 復元中は isLoading を true にして、完了後に false にする
2. 送信後の保存フローを確認・調整:
   - API 応答受信 → messages 更新 → saveMessages() 実行
   - 保存失敗時のエラーハンドリング（Toast 表示）
3. 会話が空の場合の初期表示:
   - EmptyView の設定（ガイドテキストの表示）

## タスクのチェックリスト
- [ ] 拡張機能を再起動しても前回の会話が復元される
- [ ] 新しいメッセージ送信後に永続化される
- [ ] 復元中にローディング表示がされる
- [ ] 会話が空の場合に適切な EmptyView が表示される

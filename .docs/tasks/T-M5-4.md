# T-M5-4: カスタムコマンド一覧管理コマンドの実装

| 項目 | 内容 |
|------|------|
| マイルストーン | M5 |
| 依存 | T-M5-3 |
| 要件ID | FR-015 |

## 説明
Raycast コマンド「AI Commands」を新規作成する。カスタムコマンドの一覧表示、編集、削除、および選択したコマンドでの会話開始機能を提供する。

## 作業内容
1. `src/ai-commands.tsx` を新規作成:
   - デフォルトエクスポートとして `AICommands` コンポーネントを実装
   - List コンポーネントを使用
2. `src/hooks/useCustomCommands.ts` を新規作成（必要に応じて）:
   - `commands: CustomCommand[]` - カスタムコマンド一覧
   - `isLoading: boolean` - ローディング状態
   - `addCommand()` / `updateCommand()` / `removeCommand()` - CRUD 操作
   - マウント時に `loadCustomCommands()` で復元
3. 一覧表示:
   - 各コマンドを List.Item で表示
   - `title`: コマンド名
   - `subtitle`: システムプロンプト（60文字で切り詰め表示）
   - `icon`: カスタムコマンドで設定されたアイコン（未設定時はデフォルト）
   - `accessories`: モデル名（設定時のみ表示）
4. ActionPanel:
   - Action "Start Conversation": 選択したカスタムコマンドで ask-ai チャットを開始
     - launchCommand で ask-ai を起動し、customCommandId を渡す方法を検討
     - または Action.Push で AskAI コンポーネントに customCommandId を props で渡す
   - Action.Push "Edit Command": 編集フォーム（CreateAICommand の編集版）へ遷移
     - 既存のフィールド値をフォームの初期値として設定
     - Submit 時に `updateCustomCommand()` で更新
   - Action "Delete Command"（Ctrl+X, Destructive）:
     - `confirmAlert` で確認ダイアログを表示
     - 削除時に `deleteCustomCommand()` で LocalStorage から削除
     - 成功 Toast（"カスタムコマンドを削除しました"）を表示
5. カスタムコマンドが 0 件の場合:
   - `List.EmptyView` で「カスタムコマンドがありません。Create AI Command で作成してください。」と案内
6. `package.json` の `commands` セクションにコマンドを追加:
   - `name`: "ai-commands"
   - `title`: "AI Commands"
   - `description`: "AI カスタムコマンドの一覧管理"
   - `mode`: "view"

## タスクのチェックリスト

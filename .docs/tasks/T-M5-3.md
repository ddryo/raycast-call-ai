# T-M5-3: カスタムコマンド作成コマンドの実装

| 項目 | 内容 |
|------|------|
| マイルストーン | M5 |
| 依存 | T-M5-1 |
| 要件ID | FR-014 |

## 説明
Raycast コマンド「Create AI Command」を新規作成する。Form UI でカスタムコマンドの各フィールドを入力し、LocalStorage に保存する。

## 作業内容
1. `src/create-ai-command.tsx` を新規作成:
   - デフォルトエクスポートとして `CreateAICommand` コンポーネントを実装
   - Form コンポーネントを使用
2. Form フィールド:
   - `Form.TextField` "Name"（必須）: コマンド名
   - `Form.TextArea` "System Prompt"（必須）: システムプロンプト
   - `Form.Dropdown` "Model"（任意）: モデル選択
     - 選択肢は package.json の preferences.model と同じリスト + 「デフォルト（Preferences 設定を使用）」
   - `Form.Dropdown` "Icon"（任意）: Raycast Icon からアイコンを選択
3. Submit 時の処理:
   - バリデーション: Name と System Prompt が空でないことを確認
   - UUID v4 で ID を生成
   - `addCustomCommand()` で LocalStorage に保存
   - 成功 Toast（"カスタムコマンドを作成しました"）を表示
   - `pop()` で画面を閉じる
4. `package.json` の `commands` セクションにコマンドを追加:
   - `name`: "create-ai-command"
   - `title`: "Create AI Command"
   - `description`: "AI カスタムコマンドを作成する"
   - `mode`: "view"

## タスクのチェックリスト
- [ ] コマンドが Raycast から起動できる
- [ ] 全フィールドが正しく表示・入力できる
- [ ] バリデーションが動作する（名前・プロンプト必須）
- [ ] 保存後に成功 Toast が表示される
- [ ] 保存したデータが LocalStorage に正しく永続化される
- [ ] `npm run build` でエラーなくビルドが通る

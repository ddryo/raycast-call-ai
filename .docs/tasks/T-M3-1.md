# T-M3-1: エラーハンドリングの実装

| 項目 | 内容 |
|------|------|
| マイルストーン | M3 |
| 依存 | T-M2-2 |
| 要件ID | FR-006 |

## 説明
OpenAI API のエラーを種別ごとに分類し、ユーザーにわかりやすい Toast メッセージで表示する。T-M1-3 で実装したエラー分類ロジックを UI 層に統合する。

## 作業内容
1. useConversation 内の sendMessage() にエラーハンドリングを追加
2. エラー種別ごとの Toast メッセージを定義:
   - auth（401）: 「API キーが無効です。設定を確認してください。」
   - rate_limit（429）: 「レート制限に達しました。しばらく待ってから再試行してください。」
   - timeout: 「リクエストがタイムアウトしました。再試行してください。」
   - network: 「ネットワークに接続できません。接続を確認してください。」
   - unknown: 「エラーが発生しました。再試行してください。」
3. showToast() で Style.Failure を使って表示
4. エラー発生時に isLoading を false に戻す

## タスクのチェックリスト
- [ ] 401 エラー時に API キー無効のメッセージが Toast で表示される
- [ ] 429 エラー時にレート制限のメッセージが Toast で表示される
- [ ] タイムアウト時に適切なメッセージが Toast で表示される
- [ ] ネットワーク断時に適切なメッセージが Toast で表示される
- [ ] エラー発生後も isLoading が false に戻り、操作が続行できる
